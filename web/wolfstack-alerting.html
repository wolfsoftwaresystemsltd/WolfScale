Overview

The Alerting & Notifications system monitors all nodes in your WolfStack cluster ‚Äî including the local server and every remote node ‚Äî and sends instant notifications when resource usage exceeds your configured thresholds or when nodes go offline.

Configure it from Settings ‚Üí Alerting in the WolfStack dashboard. Alerts are delivered to Discord, Slack, and/or Telegram simultaneously.

üì° Notification Channels

üìä Threshold Configuration

Set the percentage at which each metric triggers an alert. The monitor checks all nodes in your cluster ‚Äî both the local server and every remote WolfStack node.

Metric Default What It Monitors
CPU 90% Overall CPU usage percentage across all cores
Memory 90% Physical RAM usage (excluding swap)
Disk 90% Highest usage percentage among all mounted disks

Use the sliders in the Alerting tab to adjust each threshold between 1% and 100%. Metrics are evaluated against these thresholds on every check cycle.

‚è±Ô∏è Check Interval

The Check Interval controls how often WolfStack evaluates all nodes against your thresholds. Choose a balance between responsiveness and resource usage:

Interval Best For
30 seconds Critical production systems where immediate response is essential
1 minute (default) Most setups ‚Äî good balance of speed and low overhead
2 minutes Larger clusters where you want to reduce polling load
5 minutes Non-critical environments or clusters with many nodes
10 minutes Low-priority monitoring or resource-constrained nodes
üí° Note: The check interval uses metrics that are already collected by WolfStack's existing monitoring ‚Äî it does not generate extra API calls to remote nodes. The overhead is negligible at any interval.

üîî Event Types

Each event type can be individually enabled or disabled. This lets you fine-tune which alerts you receive.

üõ°Ô∏è Spam Prevention

WolfStack includes several safeguards to prevent notification floods:

‚è≥ 15-Minute Cooldown

After sending an alert for a specific metric on a specific node, the same alert won't fire again for 15 minutes ‚Äî even if the metric stays above threshold.

‚úÖ Recovery Notifications

When a metric drops back below its threshold, a recovery "OK" notification is sent and the cooldown is cleared ‚Äî ready to alert again if needed.

üëë Primary Alerter

In multi-node clusters, only the node with the lowest ID sends threshold alerts. This prevents every node from sending duplicate notifications.

üîÑ State-Change Detection

Node offline/restored alerts fire only on state transitions ‚Äî a node that stays offline generates exactly one alert, not one per polling cycle.

‚öôÔ∏è How It Works

The alerting system runs as a background task within WolfStack. Here's the monitoring flow:

  1. Startup: WolfStack waits 90 seconds after boot to let metrics stabilise
  2. Config Load: The alerting configuration is re-read from /etc/wolfstack/alerts.json on every cycle (changes take effect immediately)
  3. Node Scan: All online nodes (local + remote) are checked using their existing cached metrics ‚Äî no extra network requests
  4. Threshold Check: CPU, memory, and disk usage are compared against configured thresholds
  5. Cooldown Check: If the alert is in the 15-minute cooldown window, it's suppressed
  6. Alert Dispatch: Notifications are sent asynchronously to all configured channels
  7. Recovery Check: If a previously-alerted metric has recovered, a recovery notification is sent
  8. Sleep: Wait for the configured interval before the next cycle

üöÄ Quick Setup

  1. Go to Settings ‚Üí Alerting tab
  2. Toggle Enable Alerting on
  3. Paste your Discord/Slack webhook URL or Telegram bot credentials
  4. Click Send Test Alert to verify delivery
  5. Adjust thresholds with the sliders (default: 90% for each)
  6. Choose your preferred Check Interval
  7. Enable/disable individual event types as needed
  8. Click Save Settings ‚Äî you'll see a green ‚úì Saved! confirmation
üí° Tip: For a quick test, set the CPU threshold to a very low value (e.g. 5%) and wait for one check cycle. You should receive a threshold alert. Then set it back to your desired value and you'll get a recovery notification.
