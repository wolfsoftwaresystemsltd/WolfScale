Manage Fail2ban, UFW firewall, iptables rules, and system updates from the WolfStack dashboard

Overview

The Security page gives you a unified view of your server's security posture. Access it by selecting a server in the left sidebar, then clicking Security in that server's sub-menu. The page displays four main sections: System Updates, Fail2ban, UFW Firewall, and iptables.

For multi-server setups, each node has its own Security page ‚Äî all management actions are performed on the selected node, including remote nodes via WolfStack's proxy layer.

üì¶ System Updates

WolfStack auto-detects your package manager (apt, dnf, yum, or pacman) and shows the number of pending system updates.

üîç Check for Updates

Click Check to refresh the package cache and see how many updates are available. This runs apt-get update (or equivalent) in the background.

‚¨ÜÔ∏è Apply All Updates

Click Update All to apply all pending updates. This runs apt-get upgrade -y (or equivalent) and shows the output when complete.

üö´ Fail2ban

Fail2ban automatically bans IP addresses that show malicious behaviour (too many failed login attempts, brute-force attacks, etc.). WolfStack shows whether Fail2ban is installed, which jails are active, and which IPs are currently banned.

Installing Fail2ban

If Fail2ban is not installed, the Security page shows an Install button. Clicking it will:

  1. Install Fail2ban using your system's package manager
  2. Create a sensible default /etc/fail2ban/jail.local configuration
  3. Enable and start the Fail2ban service

jail.local Settings

Once installed, the dashboard displays the parsed settings from jail.local in a clean grid:

Setting Description Default
Ban Time How long an IP stays banned 1h
Find Time Time window for counting failures 10m
Max Retry Number of failures before banning 5
Ignore IPs IPs that are never banned (whitelist) 127.0.0.1/8 ::1

Editing jail.local

Click Edit jail.local to open a full-screen modal editor with the raw jail.local file content. Edit the configuration, click Save & Restart, and Fail2ban will automatically reload with the new settings.

/etc/fail2ban/jail.local (default)
[DEFAULT]
bantime  = 1h
findtime = 10m
maxretry = 5
ignoreip = 127.0.0.1/8 ::1

[sshd]
enabled = true

Active Jails & Banned IPs

The dashboard shows all active Fail2ban jails (e.g. sshd) and lists currently banned IP addresses. This data is fetched from fail2ban-client status.

üî• UFW Firewall

UFW (Uncomplicated Firewall) provides a user-friendly interface to manage iptables rules. WolfStack displays all current UFW rules and lets you manage them directly.

Viewing Rules

All active UFW rules are displayed in a table showing the rule number, action (ALLOW/DENY), direction (IN/OUT), and port/protocol. Rules are fetched from ufw status numbered.

Adding Rules

Click Add Rule and specify the port number (e.g. 80, 443, 8553) and the action (allow or deny). The rule is applied immediately.

Deleting Rules

Each rule has a delete button. Clicking it removes the rule from UFW. Use caution ‚Äî removing a rule that allows your SSH connection may lock you out.

‚ö†Ô∏è Important: If UFW is not installed, the Security page shows an Install & Enable button. Installing UFW will also add a rule to allow WolfStack's port (8553) to prevent lockout.

üìã iptables Rules

For servers not using UFW, the Security page displays the raw iptables rules. This is a read-only view showing the output of iptables -L -n --line-numbers, giving you visibility into the kernel's packet filtering rules.

If you need to modify iptables rules directly, use the Terminal or AI Agent features, or manage them through UFW which provides a friendlier interface.
