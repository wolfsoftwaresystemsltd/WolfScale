Docker, LXC & KVM/QEMU Virtual Machine Management with WolfStack

Overview

WolfStack provides a unified management interface for Docker containers, LXC containers, and KVM/QEMU virtual machines. You can create, start, stop, restart, clone, migrate, and monitor containers and VMs ‚Äî all from the web dashboard. If Docker or LXC isn't installed, WolfStack can install it for you with one click. QEMU/KVM is installed automatically by the setup script.

üí° Tip: WolfStack auto-detects whether Docker and LXC are installed. The container pages only appear when the corresponding runtime is available. You can install either runtime from the Containers page.

üê≥ Docker Container Management

Listing Containers

The Docker Containers page shows all containers (running and stopped) with real-time stats:

  • Name ‚Äî Container name
  • Image ‚Äî Docker image the container was created from
  • Status ‚Äî Current state (running, exited, created, paused)
  • IP Address ‚Äî Container's network IP, or WolfNet IP if configured
  • CPU ‚Äî Real-time CPU usage percentage
  • Memory ‚Äî Current memory usage (e.g. "45.2 MiB")
  • PIDs ‚Äî Number of running processes

Container Actions

Each container has these actions available from the dashboard:

‚ñ∂ Start Start a stopped container. Re-applies WolfNet IP if configured. When stopped/exited
‚èπ Stop Gracefully stop a running container When running
üîÑ Restart Stop and start the container. Re-applies WolfNet IP. When running
‚è∏ Pause Freeze all processes (SIGSTOP). Container stays in memory. When running

Creating Docker Containers

Click "Create Container" to open the creation form:

  1. Search Docker Hub ‚Äî Search for any image (e.g. "nginx", "ubuntu", "mariadb")
  2. Container Name ‚Äî Give your container a unique name
  3. Image ‚Äî Select the image and tag (e.g. "nginx:latest")
  4. Ports ‚Äî Map host ports to container ports (e.g. "8080:80")
  5. Environment Variables ‚Äî Set environment variables (e.g. "MYSQL_ROOT_PASSWORD=secret")
  6. WolfNet IP ‚Äî Optionally assign a WolfNet IP address (e.g. "10.10.10.101")
  7. Memory Limit ‚Äî Optional memory cap (e.g. "512m", "1g")
  8. CPU Limit ‚Äî Optional CPU limit (e.g. "0.5" for half a core)
Example
Name:       my-nginx
Image:      nginx:latest
Ports:      8080:80, 443:443
WolfNet IP: 10.10.10.101
Memory:     256m
CPUs:       0.5

Image Management

  • Repository & Tag: Image name and version
  • Size: Disk usage
  • Created: Creation date
  • Actions:
    • ‚ñ∂ Use: Click to immediately start creating a new container from this image. Pre-fills the Create Container form.
    • üóë Delete:Remove the image from the local system (docker rmi). Only possible if no containers are using it.

Creating LXC Containers

Click "Create Container" and select from available templates:

  1. Browse Templates ‚Äî Choose distribution (Debian, Ubuntu, Alpine, CentOS, Fedora, etc.)
  2. Select Release ‚Äî Choose the OS version (e.g. "bookworm", "jammy", "3.19")
  3. Architecture ‚Äî Usually "amd64" (auto-detected)
  4. Container Name ‚Äî Give your container a unique name
  5. Storage Path ‚Äî Optional custom storage location

The template is downloaded automatically on first use. Subsequent creations from the same template are near-instant (cached).

Editing LXC Configuration

Click the ‚öôÔ∏è Config button to open the full LXC configuration file (/var/lib/lxc/{name}/config) in an in-browser editor. Changes are saved directly and take effect on the next container start.

LXC Config Example
# Distribution configuration
lxc.include = /usr/share/lxc/config/common.conf
lxc.arch = linux64

# Container specific configuration
lxc.rootfs.path = dir:/var/lib/lxc/my-container/rootfs
lxc.uts.name = my-container

# Network configuration
lxc.net.0.type = veth
lxc.net.0.link = lxcbr0
lxc.net.0.flags = up
lxc.net.0.hwaddr = 00:16:3e:xx:xx:xx

# Resource limits
lxc.cgroup2.memory.max = 512M
lxc.cgroup2.cpu.max = 50000 100000

üñ•Ô∏è Virtual Machine Management (KVM/QEMU)

WolfStack supports full KVM/QEMU virtual machines alongside containers. VMs provide complete hardware isolation and can run any operating system, including Windows.

Creating a Virtual Machine

Click "+ Create VM" to open the creation form:

  1. Name ‚Äî Give your VM a unique name (e.g. "ubuntu-server")
  2. CPUs ‚Äî Number of virtual CPUs to allocate
  3. Memory (MB) ‚Äî RAM allocation in megabytes (e.g. 1024, 2048, 4096)
  4. Disk Size (GB) ‚Äî Virtual disk size in gigabytes (qcow2 format)
  5. ISO Path ‚Äî Optional path to an ISO image for OS installation
Example
Name:       ubuntu-server
CPUs:       2
Memory:     2048 MB
Disk:       20 GB
ISO:        /var/lib/wolfstack/isos/ubuntu-24.04-live-server-amd64.iso

VM Actions

Action Description When Available
‚ñ∂ Start Boot the VM. Allocates a VNC display for console access. When stopped
‚èπ Stop Power off the VM (force stop via process kill) When running
üóë Delete Permanently delete the VM and its disk image When stopped

VNC Console Access

Each running VM is assigned a VNC display port (5910‚Äì5999). The port is shown in the dashboard. Connect using any VNC viewer:

bash
# Connect to a VM's VNC console
vncviewer your-server-ip:5942

# Or use TigerVNC, RealVNC, Remmina, etc.

Disk Images

VM disk images are stored at /var/lib/wolfstack/vms/ in qcow2 format (QEMU Copy-On-Write). This format supports:

  • Thin provisioning ‚Äî only uses disk space as data is written
  • Snapshots ‚Äî qcow2 supports internal snapshots (future feature)
  • Compression ‚Äî can be compressed for backups
‚öôÔ∏è Requirements: QEMU/KVM is installed automatically by setup.sh. Your CPU must support hardware virtualisation (Intel VT-x or AMD-V). Check with: egrep -c '(vmx|svm)' /proc/cpuinfo (should return > 0).

üìã Cloning Containers

Docker Cloning

Cloning a Docker container:

  1. WolfStack commits the running container to a temporary image
  2. Creates a new container from that image with the new name
  3. If the original had a WolfNet IP, the clone gets a new unique IP automatically
  4. The clone is created in a stopped state ‚Äî start it when ready

LXC Cloning

LXC supports two clone methods:

Full Copy

Creates a complete, independent copy of the container's rootfs. Takes more disk space but the clone is fully independent.

Snapshot (Copy-on-Write)

Uses filesystem snapshots (btrfs/lvm). Near-instant creation, minimal disk space. Requires a supported storage backend.

üåê WolfNet IP Handling: When cloning a container that has a WolfNet IP, WolfStack automatically assigns a new unique IP from the WolfNet pool to avoid conflicts.

üöÄ Container Migration

WolfStack can migrate Docker containers between nodes in your cluster. This is useful for load balancing, maintenance, or disaster recovery.

How Migration Works

  1. Export ‚Äî WolfStack exports the container as a tar archive
  2. Transfer ‚Äî The archive is sent to the target WolfStack node over HTTP
  3. Import ‚Äî The target node imports the image and creates a new container
  4. Cleanup ‚Äî Optionally remove the source container after successful migration

Requirements

  • Both nodes must be running WolfStack
  • Both nodes must have Docker installed
  • Nodes must be able to reach each other (via WolfNet or direct IP)
  • The target node must be added to the source's cluster
üí° Tip: For containers with persistent data, pair WolfStack migration with WolfDisk for automatic data replication.

üåê WolfNet Integration

Containers can be assigned WolfNet IP addresses to make them accessible across your entire mesh network. This is configured during container creation.

How It Works

Docker Containers

  1. When you create a Docker container with a WolfNet IP, it's stored as a wolfnet.ip label
  2. When the container starts, WolfStack adds the IP as a /32 alias on eth0
  3. Routes for the 10.10.10.0/24 subnet are configured via the Docker bridge gateway
  4. Host-side iptables rules enable forwarding between wolfnet0 and docker0
  5. The WolfNet IP is automatically re-applied on container start/restart

LXC Containers

  1. WolfNet IP is saved in a marker file at /var/lib/lxc/{name}/.wolfnet/ip
  2. On container start, the IP is applied as a secondary address on eth0
  3. The container is then reachable from any WolfNet-connected machine

IP Display

When a container has a WolfNet IP configured, the dashboard shows the WolfNet IP as the primary address (with a (wolfnet) label). This IP is shown even when the container is stopped, so you always know its assigned address.

IP Allocation

WolfStack automatically suggests the next available IP in the 10.10.10.0/24 range. It checks for conflicts against:

  • The host's WolfNet IP
  • Remote cluster node IPs
  • Docker containers on the wolfnet network
  • LXC containers with WolfNet IPs

üíª Web Terminal (Console)

WolfStack includes a built-in web terminal for interactive shell access to both Docker and LXC containers ‚Äî no SSH needed.

Features

  • Full terminal emulation ‚Äî powered by xterm.js with 256-colour support
  • Monospace fonts ‚Äî JetBrains Mono, Fira Code, or Cascadia Code
  • Auto-resize ‚Äî terminal adapts to window size
  • Shell detection ‚Äî automatically uses /bin/bash if available, falls back to /bin/sh
  • WebSocket connection ‚Äî low-latency, real-time I/O

How to Use

  1. Click the üíª Console button on any running container
  2. A new browser window opens with a full terminal
  3. You're now inside the container with an interactive shell
  4. Type exit or close the window to disconnect
üîß Technical Detail: The console uses a WebSocket connection at ws://host:8553/ws/console/{type}/{name} backed by a pseudo-terminal (PTY). For Docker, this runs docker exec -i; for LXC, it uses lxc-attach.

üîß Troubleshooting

Container shows no IP address

Non-running containers don't have a network IP. If a WolfNet IP was configured, it will still be displayed even when the container is stopped. Start the container to see its bridge IP.

Console connects but can't type

This can happen if the container doesn't have a shell installed. Ensure the container has at least /bin/sh available. Minimal images (like scratch or distroless) won't support console access.

WolfNet IP not working after restart

WolfStack automatically re-applies the WolfNet IP when a container starts. If you're starting containers outside WolfStack (e.g. via docker start directly), the WolfNet IP won't be applied. Always use the WolfStack dashboard or the WolfStack API to manage containers.

LXC container won't start

Check the container config for common issues:

bash
# Check container status
lxc-info -n my-container

# Check the config file
cat /var/lib/lxc/my-container/config

# Check system logs
journalctl -u lxc@my-container -n 50
