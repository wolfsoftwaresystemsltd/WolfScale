Troubleshooting

Diagnose and resolve common WolfScale issues

Log Locations

WolfScale writes logs to standard output by default. When running as a systemd service, logs are captured by journald.

Log Type Location Command to View
Service Logs journald (systemd) journalctl -u wolfscale -f
Follow Live journald journalctl -u wolfscale -f --no-pager
Last 100 Lines journald journalctl -u wolfscale -n 100
Today's Logs journald journalctl -u wolfscale --since today
Data Directory /var/lib/wolfscale/ ls -la /var/lib/wolfscale/

Log Levels

Control log verbosity with the RUST_LOG environment variable:

Level Shows Use Case
error Errors only Production (minimal output)
warn Warnings + errors Production (recommended)
info General info + above Default level
debug Detailed debugging Troubleshooting issues
trace Everything Deep debugging (very verbose)

Setting Log Level

Command line
RUST_LOG=debug wolfscale
In systemd service
# Edit /etc/systemd/system/wolfscale.service
Environment="RUST_LOG=debug"

# Then reload
systemctl daemon-reload
systemctl restart wolfscale

Diagnostic Commands

wolfctl stats

Live dashboard showing cluster health, replication status, and throughput:

wolfctl stats

Shows: node role, LSN, replication lag, processlist, error log, and write throughput graph.

wolfctl list servers

List all nodes in the cluster with their status:

wolfctl list servers

Shows: node ID, address, role (leader/follower), status (active/syncing/lagging), and LSN.

Check WolfScale Service

# Service status
systemctl status wolfscale

# Is it running?
pgrep -a wolfscale

# Check listening ports
ss -tlnp | grep wolfscale

Check Database Connectivity

# Connect via WolfScale proxy
mysql -h 127.0.0.1 -P 3307 -u root -p

# Direct to local MariaDB
mysql -h 127.0.0.1 -P 3306 -u root -p

Common Issues

Node Won't Join Cluster

  • Check firewall: ports 7654 (cluster), 3307 (proxy), 8080 (API) must be open
  • Verify peers can reach each other: telnet <peer-ip> 7654
  • Check logs: journalctl -u wolfscale -n 50
  • Ensure databases are in sync before starting

Replication Lag Increasing

  • Check network latency between nodes
  • Monitor write throughput: wolfctl stats
  • Check if follower's MariaDB is slow (disk IO, locks)
  • Increase max_connections if needed

Connection Refused on Port 3307

  • Ensure WolfScale is running: systemctl status wolfscale
  • Check if proxy is listening: ss -tlnp | grep 3307
  • Verify the local MariaDB is accessible first

Split-Brain / Multiple Leaders

  • WolfScale uses deterministic leader election (lowest node ID wins)
  • If you see multiple leaders, check network partitioning
  • Restart nodes one at a time to resolve

Port Reference

Port Service Purpose
3306 MariaDB/MySQL Local database (direct)
3307 WolfScale Proxy MySQL protocol with read/write splitting
7654 WolfScale Cluster Inter-node replication and gossip
8080 HTTP API / Stats wolfctl, monitoring, REST API
