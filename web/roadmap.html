Roadmap

What's coming next for WolfStack and the Wolf platform

WolfStack is under active development and we have a clear vision for where the platform is heading. Below are features we're currently working on or have planned for upcoming releases. Priorities can shift based on community feedback ‚Äî if something here matters to you, let us know on Discord or Patreon.

ZFS Storage Management
Planned

Full ZFS configuration and management directly from the WolfStack dashboard. Create and manage zpools, datasets, and snapshots without touching the command line. Monitor pool health, scrub status, and disk utilisation in real time ‚Äî the same way WolfStack already handles LVM and standard partitions, but for ZFS.

  • Create, import, and destroy zpools from the UI
  • Dataset and zvol management with property editing
  • Snapshot creation, rollback, and scheduled auto-snapshots
  • Pool health monitoring with scrub and resilver status
  • Visual disk layout and vdev topology overview

PostgreSQL Support
Planned

Optional PostgreSQL support alongside the existing MariaDB/MySQL integration. Postgres brings powerful capabilities that go beyond traditional relational databases ‚Äî built-in pub/sub can replace many Redis-based caching and messaging setups, and the pgvector extension makes it a first-class vector database for AI/ML workloads, without needing a separate service. We want WolfStack users to have the choice.

  • Full PostgreSQL database browser and query editor in the dashboard
  • Leverage LISTEN/NOTIFY to reduce or eliminate Redis dependencies
  • Native pgvector support for embedding storage and similarity search
  • Replication management via WolfScale for Postgres clusters
  • One-click deployment from the App Store

Full System Backup & Restore
Planned

One-click, Time Machine-style full system backup and restore ‚Äî separate from existing data volume backups. The goal is to make bare-metal recovery as painless as restoring an iPhone from iCloud. Select a snapshot, hit restore, and your entire server comes back exactly as it was: OS, configs, services, containers, and all.

  • Incremental, deduplicated full-system snapshots
  • Browse and restore individual files or the entire system
  • Scheduled automatic backups with configurable retention
  • Independent from per-volume and per-container backups
  • Bare-metal restore to the same or replacement hardware

Cloud Storage Plugins ‚Äî pCloud & Internxt
Planned

Expanding the storage backend with first-class plugins for pCloud and Internxt Drive. These join the existing S3, NFS, and WolfDisk backends, giving you more options for offsite backup targets, cold storage, and hybrid cloud setups ‚Äî all configurable from the Storage Manager, no CLI required.

  • Mount pCloud and Internxt as storage backends in the dashboard
  • Use as backup destinations for container and system snapshots
  • Encrypted transfers with provider-side encryption support
  • Unified file browser across local and cloud storage
  • Pluggable architecture ‚Äî more providers coming based on demand

Local AI Model Support
Planned

The WolfStack AI Agent currently integrates with Claude and Gemini for intelligent server management. We're adding support for locally hosted models ‚Äî run Ollama, llama.cpp, or any OpenAI-compatible endpoint on your own hardware and use it as your AI backend. Keep your infrastructure queries entirely on-premises, reduce API costs, and maintain full data sovereignty.

  • Connect to Ollama, llama.cpp, LocalAI, or any OpenAI-compatible API
  • Run models on the same server or a dedicated GPU node in your cluster
  • Zero data leaves your network ‚Äî full privacy and sovereignty
  • Model management UI for downloading and switching between models
  • Seamless fallback ‚Äî use local models by default, cloud as backup

Centralized Log Aggregation
Planned

Aggregate logs from every node, container, and service across your cluster into a single, searchable interface. No more SSH‑ing into individual servers to tail log files ‚Äî filter by node, service, severity, or time range right from the dashboard. Built-in, lightweight, and requires zero external tooling.

  • Unified log viewer across all cluster nodes and containers
  • Full-text search with filters for service, severity, and time range
  • Live log streaming with auto-scroll and pause
  • Configurable log retention and rotation policies
  • Export and download logs for external analysis

Scheduled Tasks Manager
Planned

A visual cron and task scheduler built into the dashboard. Create, edit, and monitor scheduled jobs across your entire fleet without memorising cron syntax. See execution history, catch failures early, and run tasks on any node or group of nodes from one place.

  • Visual cron editor ‚Äî no crontab syntax required
  • Schedule tasks across specific nodes or entire clusters
  • Execution history with output capture and failure tracking
  • One-click manual execution for testing
  • Pre-built templates for common maintenance tasks

Alerting & Notifications
Planned

Configurable alerts for the things that matter ‚Äî high CPU, low disk, container crashes, node disconnects, failed backups. Set thresholds, choose your notification channels, and stop finding out about problems after your users do.

  • Threshold-based alerts for CPU, memory, disk, and network
  • Container and service health monitoring with auto-alerts
  • Notification via email, Discord webhooks, Slack, and Telegram
  • Alert history and acknowledgement tracking
  • Customisable escalation rules and quiet hours

Role-Based Access Control
Planned

Multi-user access with granular, role-based permissions. Give team members access to the nodes and features they need ‚Äî and nothing else. Ideal for agencies, teams, and anyone managing infrastructure with more than one pair of hands.

  • Predefined roles ‚Äî Admin, Operator, Viewer ‚Äî with custom options
  • Per-node and per-cluster permission scoping
  • Audit log of all user actions across the dashboard
  • SSO and LDAP/Active Directory integration
  • API key management with per-key permission sets
