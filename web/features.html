WolfScale vs Other Solutions

Aspect WolfScale Other Solutions
Replication Model Leader-based (single writer) Multi-master
Conflict Handling No conflicts Certification-based
Minimum Cluster Size 1 node 3 nodes (quorum)
Write Latency Low (local commit) Higher (sync cert)
Implementation Standalone binary Patched MariaDB

WolfScale Advantages

  • No Write Conflicts — Single leader
  • Simpler Recovery — WAL-based catch-up
  • Flexible Sizing — 1, 2, or 100 nodes
  • Lightweight — Pure Rust binary

When to Choose Other Solutions

  • True multi-master writes required
  • Cannot modify application
  • Need transparent drop-in replication

Note: In practice, most users end up writing to a single "Master Node" anyway to avoid conflicts.

Flexible Cluster Sizing

Why WolfScale Supports 2-Node Clusters

Traditional solutions like Galera require 3 nodes minimum for quorum-based consensus.

WolfScale uses deterministic leader election—lowest ID wins. No voting, no split-brain.

  • 2-node clusters work perfectly
  • Single-node valid for dev/edge
  • Scale to any size
