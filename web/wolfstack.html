Quick Install

Install WolfStack with a single command:

bash 
curl -sSL https://raw.githubusercontent.com/wolfsoftwaresystemsltd/WolfStack/master/setup.sh | sudo bash

Then open http://your-server:8553 and log in with your Linux system username and password.

What the Installer Does

  1. Detects your package manager (apt, dnf, yum)
  2. Installs build dependencies (Rust, OpenSSL, libcrypt)
  3. Clones the WolfStack repository to /opt/wolfstack-src
  4. Builds WolfStack in release mode
  5. Installs the binary to /usr/local/bin/wolfstack
  6. Installs the web UI to /opt/wolfstack/web
  7. Creates configuration at /etc/wolfstack/config.toml
  8. Creates and enables the systemd service
  9. Opens the firewall port

After Installation

bash
# View status
sudo systemctl status wolfstack

# View logs
sudo journalctl -u wolfstack -f

# Open the dashboard
xdg-open http://localhost:8553

Upgrading

Re-run the installer to upgrade. Your configuration is preserved:

Component Detail View

Click on any component card to view its detailed information:

  • Service status: Active/inactive state, sub-state (running, dead, etc.)
  • Memory usage: Current memory consumption from systemd cgroup
  • Process ID: Main PID of the service
  • Restart count: Number of times systemd has restarted the service
  • Service controls: Start, stop, restart buttons
  • Configuration editor: Full monospace editor for the config file with save button
  • Journal logs: Last 50 lines from journalctl -u <service>

Configuration

WolfStack uses a simple TOML configuration file:

toml
# /etc/wolfstack/config.toml

[server]
port = 8553
bind = "0.0.0.0"
web_dir = "/opt/wolfstack/web"

Command Line Options

bash
# Run with custom port
sudo wolfstack --port 9000

# Run with custom bind address
sudo wolfstack --bind 127.0.0.1 --port 8553
‚ö†Ô∏è Root required: WolfStack must run as root to read /etc/shadow for authentication and to manage systemd services. The setup script configures this automatically.

API Reference

All endpoints require session authentication (cookie-based) except /api/agent/status.

Method Endpoint Description
POST /api/auth/login Login with Linux credentials
POST /api/auth/logout Destroy session
GET /api/auth/check Check if session is valid
GET /api/metrics Current system metrics (CPU, memory, disk, network)
GET /api/nodes List all cluster nodes
POST /api/nodes Add a server to the cluster
DELETE /api/nodes/{id} Remove a server
GET /api/components Status of all managed components
GET /api/components/{name}/detail Component detail (config, logs, systemd info)
PUT /api/components/{name}/config Save component configuration file
POST /api/components/{name}/install Install a component
POST /api/services/{name}/action Start / stop / restart a service
POST /api/certificates Request Let's Encrypt certificate
GET /api/agent/status Node status (no auth ‚Äî used for remote polling)
