üíæ Storage Manager

Mount and manage S3, NFS, SFTP, WolfDisk, and local storage across your cluster

Overview

WolfStack's Storage Manager lets you attach external and local storage to your servers with a few clicks. Mounts are configured through the web dashboard and can be synced across your entire cluster ‚Äî so add a mount on one node and every node gets it automatically.

Under the hood, WolfStack uses rclone for cloud and remote mounts, and standard Linux mount for local filesystems. Every mount is tracked in a JSON config and managed by systemd for automatic remounting on boot.

üí° Tip: You can import your existing rclone.conf file to instantly create mounts for all your configured remotes.

üìÅ Supported Storage Types

‚òÅÔ∏è Amazon S3

Mount S3 buckets (or any S3-compatible service like MinIO, Wasabi, Backblaze B2) as local directories. Configure with access key, secret key, region, and endpoint.

üîó NFS

Mount NFS shares from any server on your network. Specify the server address and export path. Standard NFSv3/v4 with configurable options.

üîí SFTP

Mount remote directories over SSH/SFTP. Uses key-based or password authentication. Great for accessing files on remote servers securely.

üê∫ WolfDisk

Mount WolfDisk shares ‚Äî Wolf Software's own distributed filesystem. Files are replicated across all connected WolfDisk nodes in real time.

üìÇ Local Directory

Bind-mount a local directory to another path. Useful for organising storage or making directories available to containers at specific mount points.

‚ûï Creating Storage Mounts

Click "+ Add Mount" in the Storage Manager to create a new mount. The form adapts based on the storage type you select:

S3 Mount

Field Description Example
Name Friendly name for this mount backups-s3
Bucket S3 bucket name my-company-backups
Region AWS region or region of your S3-compatible service eu-west-2
Access Key Your S3 access key ID AKIA...
Secret Key Your S3 secret access key (hidden)
Endpoint Custom endpoint for S3-compatible services (leave blank for AWS) https://s3.wasabisys.com
Mount Point Where to mount on the filesystem /mnt/backups-s3

NFS Mount

Example
Name:         nas-share
Source:       192.168.1.50:/export/data
Mount Point:  /mnt/nas
Options:      vers=4,soft,timeo=30

SFTP Mount

Example
Name:         remote-server
Host:         backup.example.com
User:         admin
Key File:     /root/.ssh/id_rsa
Remote Path:  /home/admin/data
Mount Point:  /mnt/remote

WolfDisk Mount

Example
Name:         shared-files
Source:       10.10.10.1:8100
Mount Point:  /mnt/wolfdisk

Local Directory (Bind Mount)

Example
Name:         docker-data
Source:       /var/lib/docker/volumes
Mount Point:  /mnt/docker-volumes

‚öôÔ∏è Managing Mounts

Each mount in the Storage Manager table has a set of actions:

Action Description
‚ñ∂ Mount Activate the mount. For S3/SFTP this starts an rclone process; for NFS/local it runs mount.
‚èπ Unmount Deactivate the mount. Stops the rclone process or runs umount.
‚úèÔ∏è Edit Modify the mount's configuration (name, source, credentials, mount point, options).
üìã Duplicate Create a copy of this mount with a new name. Useful for creating similar mounts with slight variations.
üîÑ Sync to Cluster Push this mount configuration to all other nodes in your WolfStack cluster. Remote nodes will create and mount it automatically.
üóë Delete Remove the mount. Unmounts first if currently active.

üîÑ Cluster Storage Sync

The "Sync to Cluster" feature is one of the most powerful parts of the Storage Manager. When you sync a mount:

  1. WolfStack sends the mount configuration to every other node in the cluster
  2. Each remote node creates the mount locally
  3. Each remote node activates the mount immediately
  4. You get a summary showing which nodes succeeded or failed

This means you can configure an S3 bucket on one server and have it appear on all your servers within seconds ‚Äî no need to SSH into each machine.

üîê Security: Cluster sync uses the built-in WolfStack cluster secret for authentication. Only nodes in your cluster can receive mount configurations. Credentials (like S3 keys) are transmitted securely between nodes.

üì• Importing rclone.conf

If you already use rclone, you can import your existing configuration. Click "üì• Import rclone.conf" and paste the contents of your rclone.conf file.

WolfStack will parse the config, create a mount for each remote, and auto-generate mount points at /mnt/{remote-name}.

rclone.conf
[backups]
type = s3
provider = AWS
access_key_id = AKIA...
secret_access_key = ...
region = eu-west-2

[nas]
type = sftp
host = 192.168.1.50
user = admin
key_file = /root/.ssh/id_rsa

This would create two mounts: backups (S3) at /mnt/backups and nas (SFTP) at /mnt/nas.

üìä Status Indicators

The mount table shows real-time status for each mount:

  • ‚óè Mounted ‚Äî The mount is active and accessible at the mount point
  • ‚óè Unmounted ‚Äî The mount is configured but not currently active
  • ‚óè Error ‚Äî The mount failed (check credentials, network, or permissions)

WolfStack checks mount status by verifying the filesystem is mounted at the configured path using mountpoint -q.

üê≥ Attaching Storage to Containers

Storage mounts are integrated with WolfStack's container management. When creating or editing a Docker or LXC container, you can attach any active storage mount as a volume:

  • Storage mounts appear in the volume picker dropdown
  • The mount point path is used as the host-side path
  • Containers get immediate access to the mounted storage
Example: Docker container with S3 storage
# A container with an S3 bucket mounted at /data
Container:    my-app
Image:        nginx:latest
Volume:       /mnt/backups-s3:/data:ro

# The container can now read files from S3
# as if they were local files in /data
üí° Performance Tip: For high-throughput workloads, use NFS or local mounts. S3 and SFTP mounts work well for backups, archives, and read-heavy workloads. For real-time replication, use WolfDisk.

üìÑ Configuration

Storage mounts are stored in /etc/wolfstack/storage.json:

storage.json
[
  {
    "id": "a1b2c3d4",
    "name": "company-backups",
    "mount_type": "s3",
    "source": "my-bucket",
    "mount_point": "/mnt/backups",
    "options": "region=eu-west-2",
    "s3_access_key": "AKIA...",
    "s3_secret_key": "...",
    "s3_region": "eu-west-2",
    "s3_endpoint": ""
  }
]

You can edit this file directly, but it's recommended to use the web dashboard or API. Changes made via the API are applied immediately.

üîß Troubleshooting

Mount shows "Unmounted" but should be active

The rclone process may have crashed. Click ‚ñ∂ Mount to re-mount it. Check journalctl -u wolfstack for errors related to rclone.

S3 mount fails with "AccessDenied"

Verify your access key and secret key are correct. Ensure the IAM user/role has s3:GetObject, s3:PutObject, and s3:ListBucket permissions on the bucket.

NFS mount hangs

Check that the NFS server is reachable and the export is configured:

bash
# Check NFS exports on the server
showmount -e 192.168.1.50

# Test connectivity
rpcinfo -p 192.168.1.50

# Check firewall
nft list ruleset | grep 2049

Cluster sync fails for a node

The remote node must be online and running WolfStack. Check that the node is reachable and the cluster secret matches. View sync results in the response summary.
