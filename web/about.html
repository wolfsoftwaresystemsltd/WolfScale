üí° Why We Built This

Setting up clustering traditionally requires:

  • Complex configuration files with dozens of options
  • Deep understanding of distributed systems theory
  • Days or weeks of testing and tuning
  • Expensive enterprise support contracts
  • Database-specific solutions that don't generalize

We wanted something different. A single binary you can curl and run. Automatic discovery of peers. Predictable, deterministic failover. And the same patterns working across databases, disks, and beyond.

üîÆ The Vision

We're building towards a future where:

  • Any server can join a cluster with a single command
  • Any database can be replicated without patching or modification
  • Any storage can be shared across nodes automatically
  • Any network can discover and organize peers without central configuration
  • Failover happens automatically and predictably
