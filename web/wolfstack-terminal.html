Overview

WolfStack includes a fully interactive web terminal powered by xterm.js. Select any node from the sidebar and click Terminal to open a real shell session ‚Äî no SSH client required. The terminal connects via WebSocket and provides a native-like experience right in your browser.

‚ú® Features

Full Shell Access

Runs as root ‚Äî you have full access to the system, just like SSH. Run any command, edit files, install packages.

Remote Nodes

Terminal works on remote cluster nodes too. WolfStack proxies the WebSocket connection through the local node automatically.

Resizable

The terminal automatically resizes to fit the available space. Drag the browser window and the terminal adjusts.

Copy & Paste

Standard copy/paste works ‚Äî select text to copy, right-click or Ctrl+Shift+V to paste. Works across all browsers.

‚öôÔ∏è How It Works

  1. You select a server and click Terminal
  2. WolfStack opens a WebSocket connection to the server
  3. A bash shell is spawned as a pseudo-terminal (PTY)
  4. xterm.js renders the terminal in your browser with full ANSI support
  5. All input/output is streamed in real-time over the WebSocket

For remote nodes, the WebSocket connection is proxied through the local WolfStack instance, so you only need port 8553 open on the remote server.

The terminal requires WolfStack authentication ‚Äî you must be logged in. The WebSocket endpoint is protected by the same session cookie used by the rest of the dashboard.
