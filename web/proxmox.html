Proxmox VE Integration

Install WolfStack directly on your Proxmox nodes for full management, or connect remotely via the Proxmox REST API. No separate agent required for API-based monitoring.

Installing WolfStack on Proxmox

WolfStack can be installed directly on your Proxmox VE host, giving you the full WolfStack dashboard alongside your Proxmox environment. This lets you manage Docker containers, LXC, VMs, WolfNet, and all Wolf suite tools directly on the Proxmox node.

Quick Installation

SSH into your Proxmox node and run:

curl -sSL https://raw.githubusercontent.com/wolfsoftwaresystemsltd/WolfStack/master/setup.sh | sudo bash

Then open http://your-proxmox-ip:8553 and log in with your Linux system credentials (the same root password you use for Proxmox).

‚ÑπÔ∏è Proxmox runs Debian ‚Äî WolfStack installs cleanly on Proxmox VE hosts since they run Debian under the hood. All dependencies are handled automatically by the installer.

Adding to Your Cluster

Once WolfStack is running on your Proxmox node, you can add it to your existing WolfStack cluster using a join token:

  1. Get the join token from the Proxmox node: wolfstack --show-token
  2. On your main WolfStack dashboard, click + Add Server
  3. Select WolfStack as the node type
  4. Enter the Proxmox node's address and the join token
  5. Click Add Server

See Multi-Server Clustering for full details on join tokens and cluster setup.

Proxmox API Integration (Alternative)

If you prefer not to install WolfStack on your Proxmox hosts, you can connect remotely via the Proxmox REST API. This gives you monitoring and guest control without installing anything on the Proxmox node.

Step 1: Create an API Token in Proxmox

To connect WolfStack to your Proxmox cluster, you need to create an API Token with sufficient privileges.

  1. Log in to your Proxmox VE web interface.
  2. Go to Datacenter ‚Üí Permissions ‚Üí API Tokens.
  3. Click Add.
  4. Select a user (e.g., root@pam) and enter a Token ID (e.g., wolfstack).
  5. Important: Uncheck "Privilege Separation" if you want full admin access, or ensure the token has the required roles (see Step 2).
  6. Click Add and copy the Secret value immediately ‚Äî Proxmox shows it only once!
‚ö†Ô∏è Save the Secret! Proxmox shows the API Token Secret only once when you create it. If you lose it, you'll need to regenerate the token.

Step 2: Set API Token Permissions

The API token needs at least Sys.Audit permission to read node status, and VM.Audit + VM.PowerMgmt to list and control guests. There are three ways to set this up:

Option A: Proxmox Web UI

Easiest for most users

Option B: Command Line

Quick one-liner via SSH

Option C: Custom Role

Minimal permissions for security

Option A: Via Proxmox Web UI

  1. Go to Datacenter ‚Üí Permissions ‚Üí Add ‚Üí API Token Permission.
  2. Set Path to / (root ‚Äî applies to all nodes).
  3. Set API Token to root@pam!wolfstack.
  4. Set Role to PVEAuditor (read-only monitoring) or PVEAdmin (full VM control).
  5. Check Propagate ‚úÖ ‚Äî this ensures it applies to child resources.

Option B: Via Command Line

SSH into your Proxmox node and run:

pveum aclmod / -token 'root@pam!wolfstack' -role PVEAuditor

For full VM control (start, stop, reboot, etc.), use PVEAdmin instead:

pveum aclmod / -token 'root@pam!wolfstack' -role PVEAdmin

Option C: Minimal Custom Role

For maximum security, create a custom role with only the permissions WolfStack needs:

# Create the role
pveum role add WolfStackMonitor -privs "Sys.Audit,VM.Audit,VM.PowerMgmt,Datastore.Audit"

# Assign it to your API token
pveum aclmod / -token 'root@pam!wolfstack' -role WolfStackMonitor
Permission Purpose
Sys.Audit Read node status, CPU, memory, disk metrics
VM.Audit List VMs and containers with their resource usage
VM.PowerMgmt Start, stop, shutdown, reboot, suspend, resume guests
Datastore.Audit View storage pools and backup information

Step 3: Add Proxmox to WolfStack

  1. Open your WolfStack Dashboard.
  2. Click + Add Server in the sidebar.
  3. Select Node Type: Proxmox VE.
  4. Fill in the connection details:
    • Cluster Name ‚Äî display name for grouping in the sidebar (e.g., "Production")
    • Server Address ‚Äî IP or hostname only, no https://
    • Port ‚Äî default 8006
    • PVE Node Name ‚Äî as shown in Proxmox sidebar (e.g., pve1)
    • Token ID ‚Äî e.g., root@pam!wolfstack
    • Token Secret ‚Äî the secret from Step 1
    • TLS Fingerprint ‚Äî optional, leave blank unless you need certificate pinning
  5. Click Add Server.
‚ÑπÔ∏è Auto-Discovery: WolfStack automatically discovers all nodes in the same Proxmox cluster and adds them. You only need to add one node per cluster ‚Äî the rest are found automatically.

What You Get

Sidebar Navigation

Proxmox clusters appear in the sidebar with a server rack icon. Each cluster shows its nodes underneath, and each node shows VM and container counts. Click any node to see its metrics, or click "VMs & Containers" to view the guest listing.

Dashboard Monitoring

Proxmox nodes appear alongside your WolfStack servers in the datacenter overview with live metrics cards showing CPU, memory, disk usage, and uptime. Proxmox nodes are marked with a PVE badge and grouped by cluster.

VM & Container Management

The VMs & Containers page shows each guest as an individual card with:

  • Name, type, and VMID ‚Äî e.g., "web1 ¬∑ VM 101 ¬∑ pve1"
  • Status badge ‚Äî ‚óè running / ‚óã stopped / ‚è∏ paused
  • CPU count ‚Äî number of virtual CPUs allocated
  • Memory usage ‚Äî progress bar with percentage and absolute values
  • Disk usage ‚Äî progress bar with colour coding (green ‚Üí yellow ‚Üí red)
  • Uptime ‚Äî human-readable duration (e.g., "5d 12h")

Guest Controls

Each guest card shows action buttons based on its current state:

Guest State Available Actions
‚óè Running üîÑ Restart, ‚èπ Shutdown, ‚õî Stop, ‚è∏ Suspend (VMs only), üñ• Console

Shutdown vs Stop: "Shutdown" sends an ACPI signal for a graceful OS shutdown. "Stop" immediately powers off the guest (like pulling the plug). Use Shutdown for production systems.

Console Access

Running guests show a üñ• Console button that opens the Proxmox noVNC console directly in a new browser tab. The link points to your Proxmox web interface (https://your-pve:8006/?console=kvm&vmid=100&node=pve1) ‚Äî no additional setup is required.

Cluster Settings

Click the ‚öôÔ∏è button next to a Proxmox cluster in the sidebar to edit:

  • Cluster Name ‚Äî change the display name
  • Token ID & Secret ‚Äî update API credentials
  • TLS Fingerprint ‚Äî update or clear the certificate fingerprint

Changes apply to all nodes in the cluster at once.

API Reference

WolfStack exposes the following REST API endpoints for Proxmox integration:

Method Endpoint Description
GET /api/nodes/{id}/pve/resources List all VMs and containers on a Proxmox node
POST /api/nodes/{id}/pve/{vmid}/{action} Perform action on a guest (start, stop, shutdown, reboot, suspend, resume)
PATCH /api/nodes/{id}/settings Update node settings (token, fingerprint, cluster name)

Example: List Guests

curl -H "Cookie: session=YOUR_SESSION" \
     https://localhost:9090/api/nodes/abc123/pve/resources

Example: Start a VM

curl -X POST \
     -H "Cookie: session=YOUR_SESSION" \
     https://localhost:9090/api/nodes/abc123/pve/100/start
