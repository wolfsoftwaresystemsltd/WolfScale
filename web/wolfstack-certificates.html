View and manage SSL/TLS certificates on your servers

Overview

The Certificates page scans your server for SSL/TLS certificates and displays them in a clean table. It automatically discovers certificates in common locations like /etc/letsencrypt/live/, /etc/ssl/certs/, and other standard paths.

For each certificate, you can see the domain name, issuer, validity period, and days until expiry. Certificates nearing expiry are highlighted with warning colours.

üìã Certificate Details

Each discovered certificate shows the following information:

Field Description
Domain The Common Name (CN) or Subject Alternative Names (SANs) the certificate covers
Issuer Certificate authority that issued it (e.g. Let's Encrypt, ZeroSSL, self-signed)
Valid From Start date of the certificate's validity period
Expires Expiry date ‚Äî colour-coded: green (30+ days), amber (7‚Äì30 days), red (<7 days or expired)
Path Full filesystem path to the certificate file

üîê Let's Encrypt Integration

If Certbot is installed, WolfStack can discover and display all Let's Encrypt certificates automatically. These are typically found in /etc/letsencrypt/live/.

You can install Certbot from the App Store page with one click. Once installed, you can also set up automatic renewal using the Cron Jobs page ‚Äî there's a premade certbot renew cron job available.

Certbot Renewal Cron
# Runs daily at 5 AM ‚Äî only renews certificates expiring within 30 days
0 5 * * * certbot renew --quiet

üìù Self-Signed Certificates

WolfStack itself uses a self-signed certificate for its HTTPS connection on port 8553. This is generated automatically during installation at /etc/wolfstack/ssl/. Your browser will show a security warning for self-signed certificates ‚Äî this is normal for internal management tools.

If you want to use a proper certificate for WolfStack, you can replace the files at /etc/wolfstack/ssl/cert.pem and /etc/wolfstack/ssl/key.pem and restart the service.

üí° Tips

  • Set up a Certbot renewal cron job from the Cron Jobs page to automate certificate renewal
  • Use the Issues Scanner to detect certificates that are about to expire across your cluster
  • If you're using WolfProxy, certificates are managed through the proxy configuration interface
  • The AI Agent can check certificate expiry dates across all nodes ‚Äî just ask!
