Adding Servers to Your Cluster

Prerequisites

  • WolfStack must be installed and running on the remote server
  • The remote server must be reachable from your local WolfStack (via WolfNet or direct IP)
  • Port 8553 (default) must be accessible

Via the Dashboard

  1. Click "Servers" in the left sidebar
  2. Click "+ Add Server"
  3. Enter the server's name (friendly label) and address (IP:port or hostname:port)
  4. Click "Add"

WolfStack immediately begins polling the remote node. Within 10 seconds, you'll see its metrics and status appear in your dashboard.

Via the API

bash 
# Add a server using WolfNet IP
curl -X POST http://localhost:8553/api/nodes \
  -H "Content-Type: application/json" \
  -H "Cookie: wolfstack_session=YOUR_SESSION" \
  -d '{"name": "office-server", "address": "10.10.10.2:8553"}'

# Add using direct IP
curl -X POST http://localhost:8553/api/nodes \
  -H "Content-Type: application/json" \
  -H "Cookie: wolfstack_session=YOUR_SESSION" \
  -d '{"name": "cloud-vps", "address": "203.0.113.10:8553"}'

How Cluster Polling Works

WolfStack uses a simple polling-based architecture for cluster management:

Self-Monitoring (every 2 seconds)

Each WolfStack node collects its own metrics and component status:

  • CPU usage (per-core and average)
  • Memory usage (total, used, available)
  • Disk usage (all mounted filesystems)
  • Network interface statistics (RX/TX bytes)
  • Load averages (1m, 5m, 15m)
  • Component status for all installed Wolf components

Remote Polling (every 10 seconds)

The cluster manager polls each remote node's /api/agent/status endpoint. This endpoint does not require authentication â€” it returns basic node information needed for cluster visibility.

API Proxy â€” Remote Management

WolfStack can proxy any API call to a remote node. This means you can manage containers, services, and components on any node from your local dashboard without opening additional ports.

How It Works

Any API endpoint can be prefixed with /api/nodes/{node_id}/proxy/ to forward it to that node:

bash 
# List Docker containers on a remote node
curl http://localhost:8553/api/nodes/ws-a1b2c3d4/proxy/api/containers/docker

# Get metrics from a remote node
curl http://localhost:8553/api/nodes/ws-a1b2c3d4/proxy/api/metrics

# Restart a service on a remote node
curl -X POST http://localhost:8553/api/nodes/ws-a1b2c3d4/proxy/api/services/wolfnet/action \
  -H "Content-Type: application/json" \
  -d '{"action": "restart"}'

# Start a Docker container on a remote node
curl -X POST http://localhost:8553/api/nodes/ws-a1b2c3d4/proxy/api/containers/docker/nginx/action \
  -H "Content-Type: application/json" \
  -d '{"action": "start"}'
ğŸ”’ Authentication: You authenticate with your local WolfStack session. The proxy forwards requests to the remote node using the unauthenticated agent endpoint for reads, or direct HTTP for writes. All traffic between nodes is encrypted when using WolfNet.

Removing Servers

To remove a server from your cluster:

  1. Go to the Servers page
  2. Click the ğŸ—‘ Remove button next to the server
  3. Confirm the removal

This only removes the server from your cluster view â€” it doesn't affect the remote WolfStack instance or any of its containers.

bash 
# Remove a server by its node ID

ğŸ”§ Troubleshooting

Remote server shows as "Offline"

  • Check that WolfStack is running on the remote server: systemctl status wolfstack
  • Verify you can reach port 8553: curl http://REMOTE_IP:8553/api/agent/status
  • If using WolfNet, check the tunnel is up: ping 10.10.10.x
  • Check firewall rules: ufw allow 8553/tcp or firewall-cmd --add-port=8553/tcp

Container migration fails

  • Ensure Docker is installed on both nodes
  • Check disk space â€” the exported container image needs temporary space
  • Verify the target node is reachable from the source
  • Check WolfStack logs on both nodes: journalctl -u wolfstack -n 50

Metrics not updating

  • Remote polling runs every 10 seconds â€” wait a few moments
  • Check the browser console for JavaScript errors
  • Verify the /api/agent/status endpoint returns data
