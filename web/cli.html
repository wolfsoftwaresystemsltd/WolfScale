CLI Reference

wolfctl command-line tool for managing WolfScale clusters

Installation

The wolfctl CLI is included with WolfScale. After installation, it's available at: 

/opt/wolfscale/wolfctl

Or add to your PATH:

sudo ln -s /opt/wolfscale/wolfctl /usr/local/bin/wolfctl

Cluster Management

Command Description
wolfctl list servers Show all cluster nodes with status, role, and replication lag
wolfctl status Show local node status (ID, role, leader, cluster size)
wolfctl stats Live throughput dashboard (updates every second, Ctrl+C to exit)
wolfctl promote Request this node to become leader
wolfctl demote Step down from leader role

Setup & Migration

Command Description
wolfctl migrate --from <host:port> Copy database from another node (for adding new nodes)
wolfctl binlog-setup Auto-detect binlog position and generate config
wolfctl check-config Validate configuration file for errors
wolfctl reset --force Reset WAL and state on all nodes (DESTRUCTIVE)

Performance Tuning

Command Description
sudo wolfctl tune-mariadb Auto-tune MariaDB for optimal performance (detects RAM/CPU)
wolfctl tune-mariadb --dry-run Preview tuning changes without applying
wolfctl tune-mariadb --no-restart Apply config without restarting MariaDB

Global Options

Option Description
-c, --config <path> Path to config file (default: /etc/wolfscale/config.toml)
-e, --endpoint <url> API endpoint to connect to (overrides config)

Examples

Common Usage
# Check cluster status
wolfctl list servers

# Watch live throughput
wolfctl stats

# Auto-tune MariaDB (detects RAM and CPU)
sudo wolfctl tune-mariadb

# Add a new node to existing cluster
wolfctl migrate --from 10.0.10.111:8080

# Setup binlog replication
wolfctl binlog-setup

# Check config for errors
wolfctl check-config -c /etc/wolfscale/config.toml
